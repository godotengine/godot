44 REPLACE_SYSTEM_ALLOCATOR on POSIX causes nedalloc's functions to be called
45 malloc, free etc. instead of nedmalloc, nedfree etc. You may or may not want
46 this. On Windows it causes nedmalloc to patch all loaded DLLs and binaries
47 to replace usage of the system allocator.
48 
49 NO_NED_NAMESPACE prevents the functions from being defined in the nedalloc
50 namespace when in C++ (uses the global namespace instead).
51 
52 NEDMALLOCEXTSPEC can be defined to be __declspec(dllexport) or
53 __attribute__ ((visibility("default"))) or whatever you like. It defaults
54 to extern unless NEDMALLOC_DLL_EXPORTS is set as it would be when building
55 nedmalloc.dll.
56 
57 USE_LOCKS can be 2 if you want to define your own MLOCK_T, INITIAL_LOCK,
58 ACQUIRE_LOCK, RELEASE_LOCK, TRY_LOCK, IS_LOCKED and NULL_LOCK_INITIALIZER.
59 
60 NEDMALLOC_DEBUG can be defined to cause DEBUG to be set differently for nedmalloc
61 than for the rest of the build. Remember to set NDEBUG to disable all assertion
62 checking too.
63 
64 USE_MAGIC_HEADERS causes nedalloc to allocate an extra three sizeof(size_t)
65 to each block. nedpfree() and nedprealloc() can then automagically know when
66 to free a system allocated block. Enabling this typically adds 20-50% to
67 application memory usage.
68 
69 ENABLE_TOLERANT_NEDMALLOC is automatically turned on if REPLACE_SYSTEM_ALLOCATOR
70 is set or the Windows DLL is being built. This causes nedmalloc to detect when a
71 system allocator block is passed to it and to handle it appropriately. Note that
72 without USE_MAGIC_HEADERS there is a very tiny chance that nedmalloc will segfault
73 on non-Windows builds (it uses Win32 SEH to trap segfaults on Windows and there
74 is no comparable system on POSIX).
75 
76 USE_ALLOCATOR can be one of these settings (it defaults to 1):
77  0: System allocator (nedmalloc now simply acts as a threadcache).
78  WARNING: Intended for DEBUG USE ONLY - not all functions work correctly.
79  1: dlmalloc
80 
81 ENABLE_LARGE_PAGES enables support for requesting memory from the system in large
82 (typically >=2Mb) pages if the host OS supports this. These occupy just a single
83 TLB entry and can significantly improve performance in large working set applications.
84 
85 ENABLE_FAST_HEAP_DETECTION enables special logic to detect blocks allocated
86 by the system heap. This avoids 1.5%-2% overhead when checking for non-nedmalloc
87 blocks, but it assumes that the NT and glibc heaps function in a very specific
88 fashion which may not hold true across OS upgrades.
89 */
