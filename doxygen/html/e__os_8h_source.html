409  /* We need to do this since VMS has the following coding on status codes:
410 
411  Bits 0-2: status type: 0 = warning, 1 = success, 2 = error, 3 = info ...
412  The important thing to know is that odd numbers are considered
413  good, while even ones are considered errors.
414  Bits 3-15: actual status number
415  Bits 16-27: facility number. 0 is considered "unknown"
416  Bits 28-31: control bits. If bit 28 is set, the shell won't try to
417  output the message (which, for random codes, just looks ugly)
418 
419  So, what we do here is to change 0 to 1 to get the default success status,
420  and everything else is shifted up to fit into the status number field, and
421  the status is tagged as an error, which I believe is what is wanted here.
422  -- Richard Levitte
423  */
424 # define EXIT(n) do { int __VMS_EXIT = n; \
425  if (__VMS_EXIT == 0) \
426  __VMS_EXIT = 1; \
427  else \
428  __VMS_EXIT = (n << 3) | 2; \
429  __VMS_EXIT |= 0x10000000; \
430  exit(__VMS_EXIT); } while(0)
