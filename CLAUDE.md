# Godot Engine - Agent Coding Guide

> Detailed reference docs in `.claude/docs/`. This file is the quick-reference cheat sheet.

## Build

```bash
scons platform=windows          # Build editor (debug)
scons platform=windows target=template_release  # Export template
scons platform=windows tests=yes && bin/godot.windows.editor.x86_64.exe --test  # Run tests
```

Build system: **SCons** (Python). Each directory has `SCsub` files. Modules have `config.py`.

## Critical Rules

| Rule | Details |
|------|---------|
| **Never `new`/`delete`** | Use `memnew(T)` / `memdelete(ptr)`. Arrays: `memnew_arr(T, count)` / `memdelete_arr(ptr)` |
| **Never STL containers** | Use `Vector<T>`, `List<T>`, `HashMap<K,V>`, `HashSet<T>`, `LocalVector<T>` |
| **Never `std::string`** | Use `String`. For interned identifiers use `StringName` |
| **Never `printf`** | Use `print_line()`, `ERR_PRINT()`, `WARN_PRINT()` |
| **Never hand-edit `.gen.*`** | Generated by Python scripts during build. Edits will be overwritten |
| **Never bare `if/return` for errors** | Use `ERR_FAIL_*` macros (see [conventions.md](.claude/docs/conventions.md)) |
| **Always `GDCLASS` macro** | Every class exposed to scripting needs `GDCLASS(MyClass, Parent)` in header |
| **Always `_bind_methods()`** | Every GDCLASS needs `static void _bind_methods()` in the `.cpp` |
| **Always update doc XML** | New/changed methods/properties need `doc/classes/ClassName.xml` updates |

## Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Classes | `PascalCase` | `Node3D`, `MeshInstance3D`, `RenderingServer` |
| Files | `snake_case` | `node_3d.h`, `mesh_instance_3d.cpp` |
| Methods | `snake_case` | `get_transform()`, `set_visible()`, `is_inside_tree()` |
| Parameters | `p_` prefix | `p_name`, `p_enabled`, `p_child` |
| Return-by-ref params | `r_` prefix | `r_error`, `r_vertex_array` |
| Private methods | `_` prefix | `_update_process()`, `_validate_child_name()` |
| Constants/Enums | `SCREAMING_SNAKE` | `NOTIFICATION_READY`, `PROCESS_MODE_INHERIT` |
| Bool getters | `is_` or `has_` | `is_visible()`, `has_node()` |

## File Structure

```
Header (.h):
  Copyright block -> #pragma once -> includes -> forward decls -> class { GDCLASS(...); ... }

Source (.cpp):
  #include "own_header.h"
  #include "own_header.compat.inc"  // if exists
  STATIC_ASSERT_INCOMPLETE_TYPE(...)  // if forward decls used
  #include "core/..."               // core includes (alphabetical)
  #include "scene/..."              // scene/server includes (alphabetical)
  #ifdef TOOLS_ENABLED              // conditional includes last
  #include "editor/..."
  #endif
```

## The Binding Checklist

When adding or modifying a class exposed to scripting:

- [ ] `GDCLASS(MyClass, ParentClass)` macro in header
- [ ] `_bind_methods()` with `ClassDB::bind_method()` for each method
- [ ] `D_METHOD("name", "arg1", "arg2")` with string names matching exactly
- [ ] `DEFVAL(value)` for optional parameters
- [ ] `ADD_PROPERTY(PropertyInfo(...), "set_x", "get_x")` for each property
- [ ] `ADD_GROUP` / `ADD_SUBGROUP` for inspector organization
- [ ] `ADD_SIGNAL(MethodInfo("signal_name"))` for each signal
- [ ] `BIND_ENUM_CONSTANT(ENUM_VALUE)` for each enum exposed to scripts
- [ ] `GDVIRTUAL` + `GDVIRTUAL_BIND` for script-overridable methods
- [ ] `doc/classes/ClassName.xml` documentation
- [ ] `VARIANT_ENUM_CAST(MyClass::MyEnum)` outside the class for enum types
- [ ] `.compat.inc` if changing existing method signatures

## Error Handling Quick Reference

```cpp
// In void functions:
ERR_FAIL_COND(condition);                    // return on error
ERR_FAIL_COND_MSG(condition, "message");     // with message (preferred)
ERR_FAIL_NULL(ptr);                          // null check
ERR_FAIL_INDEX(idx, size);                   // bounds check

// In functions with return values:
ERR_FAIL_COND_V(condition, retval);
ERR_FAIL_COND_V_MSG(condition, retval, "message");
ERR_FAIL_NULL_V(ptr, retval);
ERR_FAIL_INDEX_V(idx, size, retval);

// In loops:
ERR_CONTINUE(condition);                     // skip iteration
ERR_BREAK(condition);                        // break loop

// Severity levels:
WARN_PRINT("msg");      // Warning only, no return
ERR_PRINT("msg");       // Error only, no return
CRASH_NOW_MSG("msg");   // Fatal, only for unrecoverable errors
DEV_ASSERT(condition);  // Debug builds only
```

## Memory Management Decision Tree

```
Is it a RefCounted subclass (Resource, Script, etc.)?
  YES -> Use Ref<T>. Pass by value or const ref.
  NO  -> Is it a Node?
    YES -> Parent owns it. Use memnew() to create, queue_free() to destroy.
    NO  -> Use memnew()/memdelete() directly. You manage lifetime.
```

## Signal Connections (C++)

```cpp
// Connect with callable_mp (preferred — type-safe):
mesh->connect_changed(callable_mp(this, &MyNode::_mesh_changed));
obj->connect(SNAME("signal_name"), callable_mp(this, &MyNode::_handler));

// Disconnect (always check first):
if (mesh.is_valid()) {
    mesh->disconnect_changed(callable_mp(this, &MyNode::_mesh_changed));
}

// Deferred call (thread-safe, runs on main thread next frame):
callable_mp(this, &MyNode::_do_work).call_deferred();

// Bind extra arguments:
obj->connect(SNAME("changed"), callable_mp(this, &MyNode::_on_changed).bind(extra_arg));
```

See [runtime-patterns.md](.claude/docs/runtime-patterns.md) for full signal lifecycle, dynamic properties, undo/redo, and type casting patterns.

## Type Casting

```cpp
// Always use Object::cast_to, never dynamic_cast
if (Node3D *n3d = Object::cast_to<Node3D>(p_object)) {
    n3d->set_position(Vector3());
}
// Or with error macro:
Node3D *n3d = Object::cast_to<Node3D>(p_object);
ERR_FAIL_NULL(n3d);
```

## Thread Safety

- Node operations in scene tree: **main thread only**
- Use `callable_mp(obj, &Class::method).call_deferred()` from other threads
- Use `MutexLock lock(mutex)` for RAII locking (never manual lock/unlock)
- Use `RWLockRead` / `RWLockWrite` for read-heavy shared data
- Guard macros: `ERR_THREAD_GUARD`, `ERR_MAIN_THREAD_GUARD`, `ERR_READ_THREAD_GUARD`

## Code Style

- **Indentation**: Tabs (4-space width). Python files use spaces.
- **Line endings**: LF (except `.csproj`/`.sln`/`.bat` use CRLF)
- **Header guards**: `#pragma once` (not `#ifndef`)
- **Braces**: Always required (enforced by clang-format `InsertBraces: true`)
- **Column limit**: None enforced (ColumnLimit: 0), but .editorconfig suggests 120
- **Formatting**: Run `clang-format` before committing
